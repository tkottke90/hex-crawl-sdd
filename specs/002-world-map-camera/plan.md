# Implementation Plan: World Map Camera Behavior

**Branch**: `002-world-map-camera` | **Date**: 2026-04-21 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/002-world-map-camera/spec.md`

## Summary

Add camera behavior to the existing `WorldMap` Phaser scene: initial centering on the active
character's tile at load, smooth ease-out follow tweens when the character moves or the active
character changes, independent keyboard pan (Arrow / WASD) at 5 tiles/sec, clamped map
boundaries, and a persistent DOM re-center button in the bottom-right corner. Zoom is out of
scope. Built on top of feature `001-hex-crawl-game` using TypeScript + Phaser 3 + Tailwind v4.

## Technical Context

**Language/Version**: TypeScript 5.7 (strict mode)  
**Primary Dependencies**: Phaser 3.60+ (cameras, tweens, input), Vite 6, Tailwind v4  
**Storage**: N/A (camera state is transient; not persisted)  
**Testing**: Vitest (unit: camera controller logic) + Playwright (e2e: visual centering, pan)  
**Target Platform**: Desktop browser — Chrome, Firefox, Safari, Edge (latest 2 versions)  
**Project Type**: Browser game (single-page, no server)  
**Performance Goals**: Camera pan/follow must not drop frame rate below 60 fps; keyboard pan begins within one frame of key press  
**Constraints**: Fully offline-capable; camera state is ephemeral (not saved to IndexedDB)  
**Scale/Scope**: Single `WorldMap` scene; no new scenes; additive change to existing module

### CameraController Module Interface

```typescript
// src/modules/camera/CameraController.ts

export interface CameraControllerOptions {
  camera: Phaser.Cameras.Scene2D.Camera;
  /** Tile lookup: returns world-space pixel centre of a HexCoord. */
  tileWorldPos: (coord: HexCoord) => { x: number; y: number };
  /** Phaser TILE_SIZE in pixels. */
  tileSize: number;
}

export class CameraController {
  constructor(options: CameraControllerOptions);

  /** Call once after scene tiles are rendered. Clamps camera to map bounds. */
  setBounds(tiles: HexTile[]): void;

  /** Center camera instantly on a world coordinate (used at scene init). */
  centerOn(x: number, y: number): void;

  /** Smoothly pan to a tile with proper duration scaling. */
  followTo(coord: HexCoord, pathLength: number): void;

  /** Re-center on a coord using a fixed 300 ms ease-out tween. */
  reCenterOn(coord: HexCoord): void;

  /** Called each frame from WorldMap.update(). Handles keyboard pan. */
  update(keys: CameraKeys, delta: number): void;

  /** True while any pan key is held (free-pan mode). */
  get isFreePanActive(): boolean;
}

export interface CameraKeys {
  up: boolean;
  down: boolean;
  left: boolean;
  right: boolean;
}
```

### WorldMap.ts Integration Points

1. **`create()`** — after `renderMap()`:
   - Construct `CameraController`, call `setBounds(tiles)`, call `centerOn(activeCharWorldPos)`.
   - Guard: `centerOn` is called only after `buildParty()` completes and `selectedChar` is
     non-null; if null (should not occur under normal flow), skip centering silently.
   - Render re-center button DOM element.
2. **`onTileClick()`** — after `moveOccupant()`:
   - Call `cameraController.followTo(dest, pathLength)`.
3. **`update()`** (new method on scene) — each frame:
   - Call `cameraController.update(keys, delta)`.
4. **Re-center button click handler**:
   - Call `cameraController.reCenterOn(activeCharCoord)`.
5. **`shutdown()`** — existing teardown method:
   - Remove the re-center button DOM element (same pattern as `saveBar` teardown: `reCenterBtn.remove(); reCenterBtn = null`).
   - The `CameraController` instance is GC'd with the scene; no explicit dispose needed.

### Coordinate Refactor Scope & Rollback Criteria (R-002)

The existing `WorldMap.ts` adds `this.scale.width / 2` and `this.scale.height / 2` to every tile
and sprite pixel position in `renderMap()` and `renderParty()`. This must be removed so that
objects render at raw `toPixel()` world coordinates and the Phaser camera handles viewport
positioning.

**Files changed**:
- `src/game/scenes/WorldMap.ts` — `renderMap()` and `renderParty()`, plus the hit-area `offsetX/offsetY` additions inside `renderMap()`.

**Rollback criteria** — revert R-002 if any of the following occur after the change:
- Tiles render off-screen or at incorrect positions in the `smoke.spec.ts` e2e test.
- The `#btn-save-game` element is no longer visible in Playwright (save-bar uses DOM positioning, unaffected, but verifies the scene loaded correctly).
- Unit tests in `tests/unit/hex-grid/` fail due to unexpected pixel coordinate values.

**Risk mitigation**: The refactor must be done as a single atomic commit before any camera code is added, with the full existing test suite run to establish a green baseline before camera features are built on top.
